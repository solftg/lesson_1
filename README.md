# Теоретическая справка
Исключения в Python — это механизм обработки ошибок и особых ситуаций, возникающих во время выполнения программы. Обработка исключений позволяет писать код, устойчивый к неожиданным ошибкам и способный корректно реагировать на нестандартные ситуации.

## Основные концепции

### Что такое исключение?

**Исключение** — это объект, представляющий ошибку или необычное состояние, которое мешает нормальному выполнению программы. Когда возникает ошибка, интерпретатор генерирует объект класса Exception (или подкласса).

Например, деление на ноль вызывает исключение типа **ZeroDivisionError**.

### Зачем нужны исключения?

Обработка исключений помогает сделать программу надежной и удобной для пользователей. Например, если программа сталкивается с проблемой чтения файла, исключение позволит информативно сообщить об этом и предложить возможные пути решения проблемы, а не завершится аварийно.

### Как обрабатывать исключения?

Для обработки исключений используется конструкция try, except, else и finally. Рассмотрим подробнее каждую часть конструкции:
```
try:
    # Код, который потенциально может вызвать исключение
    result = 1 / int(input("Введите число: "))
    
except ValueError as e:
    print(f"Введено некорректное значение: {e}")
    
except ZeroDivisionError as e:
    print(f"Произошла ошибка деления на ноль: {e}")
    
else:
    print(f"Результат вычисления: {result}")
    
finally:
    print("Завершение блока try")
```
#### try: — основной блок, внутри которого выполняется потенциально опасный код.
#### except ExceptionType:— обработчики конкретных типов исключений. Если в блоке try возникло исключение указанного типа, оно обрабатывается в соответствующем блоке except.
#### else: — необязательная секция, выполняемая, если в блоке try не было возбуждено ни одно исключение.
#### finally: — этот блок выполнится вне зависимости от того, произошло исключение или нет. Полезно для очистки ресурсов, закрытия файлов и соединений.

### Основные типы исключений

Python имеет встроенную иерархию классов исключений. Вот некоторые часто используемые классы:

* **ValueError:** Неправильный аргумент функции.
* **TypeError:** Ошибка несоответствия типов аргументов функций.
* **IndexError:** Попытка обращения к несуществующему индексу списка или строки.
* **KeyError:** Обращение к несуществующему ключу в словаре.
* **FileNotFoundError:** Файл не найден.
* **IOError:** Проблемы ввода-вывода.
* **ZeroDivisionError:** Деление на ноль.

Также возможно создавать собственные классы исключений путем наследования от базового класса Exception.

## Примеры практических применений

### Безопасное чтение файла:
```
try:
    with open('file.txt', 'r') as f:
        data = f.read()
        
except FileNotFoundError:
    print("Файл не найден.")
    
except IOError:
    print("Ошибка чтения файла.")
    
else:
    print(data)
```

### Проверка правильности введенных данных:
```
while True:
    try:
        age = int(input("Введите ваш возраст: "))
        if age <= 0:
            raise ValueError("Возраст должен быть положительным числом.")
            
    except ValueError as ve:
        print(ve)
        
    else:
        break
print(f"Ваш возраст: {age}.")

```

## Best practice
* Используйте специфичные исключения (ZeroDivisionError, FileNotFoundError), а не общий класс Exception, чтобы избежать случайной маскировки серьезных проблем.
* Избегайте пустых блоков except: обязательно включайте сообщение или инструкцию для дальнейшей реакции.
* Никогда не подавляйте исключения без причины — лучше выводите осмысленные сообщения или логируйте проблему.
* Для сложных операций полезно обернуть весь процесс в конструкцию with, чтобы автоматически освобождать ресурсы (тема следующих лекций).
* Создавайте собственные исключения, когда стандартные недостаточно ясно отражают ситуацию вашей задачи.
